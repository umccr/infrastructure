# Data export SQL files

The recommendation from AWS docs is to explicitly list all columns in data export
SQL (rather than use "*"). This ensures stability of the reports as columns
may evolve over time (from the AWS side).

These queries have been autogenerated from the AWS Console UI (Feb 2026).

(note that for CUR2.0 we have taken the recommendation of AWS and removed the
resource_tags and cost_category fields as they are now completely redundant -
all the same data is in the tags field)

The queries should occasionally be checked against the default autogenerator to
see if there are any new fields or syntax changes recommended by AWS (or read the docs!).
